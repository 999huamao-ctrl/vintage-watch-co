## 2. 账号体系与权限

### 2.1 角色权限矩阵

| 角色 | 仪表盘 | 商品管理 | 订单管理 | 用户管理 | 钱包设置 | 库存管理 | 物流管理 |
|------|--------|----------|----------|----------|----------|----------|----------|
| **SUPERADMIN** | ✅ | ✅(全) | ✅(全) | ✅ | ✅ | ✅ | ✅ |
| **ADMIN** | ✅ | ✅(全) | ✅(全) | ❌ | ❌ | ✅ | ❌ |
| **SUPPLY** | ✅ | ✅(仅库存) | ❌ | ❌ | ❌ | ✅ | ❌ |
| **LOGISTICS** | ✅ | ❌ | ✅(仅物流) | ❌ | ❌ | ❌ | ✅ |

### 2.2 各角色详细权限

#### SUPERADMIN (超级管理员)
- **全系统访问权限**
- 钱包配置管理 (L1/L2/L3 USDT地址设置)
- 用户账号创建/删除/权限修改
- 查看财务报表
- 系统级配置

#### ADMIN (运营管理员)
- 仪表盘数据查看
- 商品全功能管理 (增删改查、价格调整)
- 订单全功能管理 (状态更新、退款处理)
- 库存管理 (库存调整、低库存预警)
- **无权限**: 钱包设置、用户管理

#### SUPPLY (供应链专员)
- 仪表盘数据查看
- 商品查看 (不能改价格、不能删除)
- 库存管理 (仅能修改库存数量)
- **无权限**: 订单管理、钱包设置

#### LOGISTICS (物流专员)
- 仪表盘数据查看
- 订单查看
- 物流管理 (更新发货状态、填写物流单号)
- **无权限**: 商品管理、库存管理、价格相关

### 2.3 登录方式

**管理后台地址**: https://horizon-watch-admin.vercel.app/admin

**首次登录流程**:
1. 输入用户名和密码
2. 登录成功后进入对应权限的仪表盘
3. **首次登录后必须重置密码** (系统已自动生成随机密码)

## 3. 商品管理 (SUPPLY/ADMIN/SUPERADMIN)

### 3.1 商品字段说明

| 字段 | 说明 | 可编辑角色 |
|------|------|------------|
| **Name** | 商品名称 (英文) | ADMIN/SUPERADMIN |
| **Name Zh** | 商品名称 (中文) | ADMIN/SUPERADMIN |
| **Price** | 售价 (€) | ADMIN/SUPERADMIN |
| **Original Price** | 原价/划线价 (€) | ADMIN/SUPERADMIN |
| **Stock** | 库存数量 | ADMIN/SUPERADMIN/SUPPLY |
| **Category** | 分类 | ADMIN/SUPERADMIN |
| **Status** | 上架/下架状态 | ADMIN/SUPERADMIN |
| **Description** | 商品描述 (英文) | ADMIN/SUPERADMIN |
| **Description Zh** | 商品描述 (中文) | ADMIN/SUPERADMIN |
| **Case Size** | 表壳尺寸 | ADMIN/SUPERADMIN |
| **Movement** | 机芯类型 | ADMIN/SUPERADMIN |
| **Strap** | 表带材质 | ADMIN/SUPERADMIN |
| **Water Resistance** | 防水等级 | ADMIN/SUPERADMIN |

### 3.2 SUPPLY 角色操作指南

**只能修改库存**:
1. 进入 "Products" 页面
2. 点击商品行的 "Edit" 按钮
3. 只有 "Stock" 字段可编辑 (其他字段灰色禁用)
4. 修改库存数量
5. 点击 "Save" 保存

**库存预警**:
- 库存 < 10 时显示红色警告
- 仪表盘显示 "Low Stock Products" 数量

### 3.3 ADMIN/SUPERADMIN 操作

**添加新商品**:
1. 点击 "Add Product" 按钮
2. 填写完整商品信息
3. 设置合理的库存数量
4. 点击 "Save" 保存

**删除商品**:
1. 点击商品行的 "Delete" 按钮 (垃圾桶图标)
2. 确认删除

## 4. 订单管理 (LOGISTICS/ADMIN/SUPERADMIN)

### 4.1 订单状态流转

```
PENDING (待支付) 
    ↓ 支付完成
PAID (已支付)
    ↓ 确认订单
CONFIRMED (已确认)
    ↓ 开始处理
PROCESSING (处理中)
    ↓ 发货
SHIPPED (已发货)
    ↓ 送达
DELIVERED (已送达)

其他状态:
- CANCELLED (已取消)
- REFUNDED (已退款)
```

### 4.2 LOGISTICS 角色操作

**更新订单状态**:
1. 进入 "Orders" 页面
2. 点击订单查看详情
3. 在 "Update Status" 区域操作:
   - 选择新状态
   - 填写物流单号 (Tracking Number)
   - 选择承运商 (Carrier)
4. 点击 "Update" 保存

**物流信息填写**:
- **Tracking Number**: 物流追踪号码
- **Carrier**: 物流公司名称
- 客户会收到包含物流信息的邮件通知

### 4.3 ADMIN/SUPERADMIN 操作

**处理退款**:
1. 打开订单详情
2. 状态选择 "REFUNDED"
3. 系统记录退款状态
4. **注意**: 实际的 USDT 退款需要手动在钱包操作

**取消订单**:
1. 订单状态改为 "CANCELLED"
2. 只有 PAID 之前的订单可以取消

## 5. 钱包与收款管理 (SUPERADMIN ONLY)

### 5.1 3级钱包架构

```
┌─────────────────────────────────────────────────────────────┐
│ Level 1 (收款钱包)                                          │
│ 地址: TGPBhfjSuwjrfGUtdqt6EZUbzhbRCGfC5c                    │
│ 功能: 仅接收客户 USDT 付款                                  │
│ 操作: 每日归集资金到 L2                                     │
├─────────────────────────────────────────────────────────────┤
│ Level 2 (运营钱包)                                          │
│ 地址: TCWgr2qGcheRsD7kceoFpJfMg59fFrJGCq                    │
│ 功能: 广告费、物流费、运营成本                              │
│ 分配: 收款金额的 40%                                        │
├─────────────────────────────────────────────────────────────┤
│ Level 3 (利润钱包)                                          │
│ 地址: TUyTqV47pd7o3Bg6Uhw5XJ9rwkdgi6tsKb                    │
│ 功能: 最终利润存储                                          │
│ 分配: 收款金额的 60%                                        │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 资金归集流程

**每日操作**:
1. 检查 L1 收款钱包余额
2. 确认客户付款与订单匹配
3. 资金分配:
   - 40% → L2 运营钱包
   - 60% → L3 利润钱包
4. 记录归集交易哈希

### 5.3 收款确认流程

1. 客户选择 USDT 支付
2. 系统显示收款地址 (L1)
3. 客户完成转账
4. 在订单管理中找到对应订单
5. 确认收款后，订单状态改为 "CONFIRMED"
6. 执行资金归集

## 7. 物流渠道信息

### 7.1 欧洲敏感专线

| 项目 | 信息 |
|------|------|
| **承运商** | 敏感专线-欧美-通 |
| **覆盖国家** | 德/法/意/西/葡/荷/波/捷/英等 17 国 |
| **时效** | 9-15 工作日 |
| **单只运费** | €5-8 (0.3-0.5kg) |

### 7.2 运费计算

```
总运费 = 处理费 (16-35元) + 运费 (65-85元/kg) × 重量
单只手表 ≈ €5-8
```

### 7.3 物流状态更新

1. 发货后获取物流单号
2. 在订单详情中填入:
   - Tracking Number (追踪号码)
   - Carrier (承运商名称)
3. 客户自动收到物流通知邮件

## 8. 日常运营 SOP

### 8.1 每日检查清单

**SUPPLY 每日任务**:
- [ ] 检查仪表盘 "Low Stock Products"
- [ ] 确认库存充足的商品
- [ ] 更新缺货商品库存

**LOGISTICS 每日任务**:
- [ ] 查看 "Pending Orders" 列表
- [ ] 处理 PAID 状态的订单 (确认并安排发货)
- [ ] 更新 SHIPPED 订单的物流信息

**ADMIN 每日任务**:
- [ ] 检查今日收入和订单数
- [ ] 处理退款/取消订单
- [ ] 检查异常订单

**SUPERADMIN 每日任务**:
- [ ] 检查 L1 收款钱包余额
- [ ] 确认 USDT 收款并归集资金
- [ ] 检查 L2 运营钱包余额 (广告/物流备用金)

### 8.2 异常处理

**订单异常**:
| 情况 | 处理方式 |
|------|----------|
| 客户未付款 | 24小时后自动取消 |
| 付款金额不符 | 联系客户补差或退款重付 |
| 地址信息错误 | 联系客户确认，必要时取消订单 |
| 缺货 | 联系客户换款或退款 |

**物流异常**:
| 情况 | 处理方式 |
|------|----------|
| 包裹退回 | 检查地址，重新发货或退款 |
| 物流延迟 | 主动联系客户说明情况 |
| 包裹丢失 | 联系物流商索赔，给客户退款或补发 |
