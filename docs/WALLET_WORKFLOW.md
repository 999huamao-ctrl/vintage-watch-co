# 2级钱包收款与资金流转流程

## 钱包架构

```
客户付款
    ↓
Level 1: 收款钱包 (仅接收)
    TGPBhfjSuwjrfGUtdqt6EZUbzhbRCGfC5c
    ↓ 收到订单后手动/定期转入
Level 2: 运营钱包 (支出用途)
    TCWgr2qGcheRsD7kceoFpJfMg59fFrJGCq
    ↓ 每周/每月汇总后转入
Level 3: 利润钱包 (最终归属)

```

## 钱包用途说明

| 层级 | 地址 | 用途 | 操作权限 |
|------|------|------|----------|
| L1 收款 | TGPBhfj... | 仅用于网站展示，接收客户付款 | S1查看，不主动操作 |
| L2 运营 | TCWgr2q... | 广告费、物流费、网站运维、应急资金 | S1全权管理 |
| L3 利润 | TUyTqV47... | 最终利润归集，定期提现 | STAR持有 |

## 资金流转规则

### 收入分成
- **40% 再投入**: 从L2转入广告/运营支出
- **60% 利润**: 从L1直接转入L3（或经L2汇总）

### 操作流程

#### 1. 客户下单付款
- 客户在网站下单，看到USDT收款地址（L1）
- 客户转账后点击"已完成付款"
- 系统记录订单状态为"待确认"

#### 2. 收款确认 (S1操作)
1. 每2-4小时检查L1钱包余额
2. 核对订单金额与到账金额（允许±5%波动）
3. 匹配交易哈希(TXID)与订单
4. 确认后更新订单状态为"已付款"

#### 3. 资金归集 (S1操作)
**每日归集流程:**
```bash
# 从L1转入L2（运营资金）
金额 = 当日收款 × 40%
目的: 广告费、物流费储备

# 从L1转入L3（利润）  
金额 = 当日收款 × 60%
目的: 利润归集
```

**操作示例:**
- 今日收款: 5,000 USDT
- 转入L2运营: 2,000 USDT
- 转入L3利润: 3,000 USDT

#### 4. 运营支出 (S1操作)
从L2钱包支付:
- Facebook/Google广告充值
- 物流运费（如提前支付给货代）
- 网站域名/服务器续费
- 其他运营开支

## 订单管理系统

### 订单状态流转

```
待付款 → 已付款 → 待发货 → 已发货 → 已完成
            ↓
        已取消 (超时未付款)
```

## 安全注意事项

1. **地址确认**: 每次转账前三次核对地址前缀和后缀
2. **网络确认**: 仅使用TRC20网络，禁止其他网络
3. **小额测试**: 首次转账新地址先转1 USDT测试
4. **私钥保管**: L2钱包私钥由S1保管，定期备份

## 紧急情况处理

| 情况 | 处理方案 |
|------|----------|
| 客户付错金额 | 联系客户补款或退款重付 |
| 客户付错网络 | 尝试找回，如无法找回协商处理 |
| L1钱包异常 | 立即启用备用地址，更新网站 |
| L2资金不足 | 从L1紧急调入，记录后补 |
